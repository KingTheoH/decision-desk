PROPERTY ESTIMATION RULES — critical, always follow for Property type:
- monthly_rent_estimate: OUTPUT IN USD ONLY. Convert local rents to USD first (¥150 = $1 USD). Rural Japan: ¥40,000-70,000/mo = $267-467/mo USD. Suburban Japan: ¥80,000-150,000/mo = $533-1000/mo USD. Central Osaka/Tokyo: ¥150,000-350,000/mo = $1000-2333/mo USD. A rural Fukushima house should be around $300-500/mo USD. NEVER set monthly_rent_estimate close to the property value — a property worth $190,000 does NOT rent for $8,000/month.
- gross_yield_pct: Always calculate — (monthly_rent_estimate * 12 / asking_price) * 100. For rural Japan typical range is 3-7%. Never leave null.
- expected_return: Net yield = gross_yield_pct minus ~2.5% for fees, tax, vacancy. Set as top-level expected_return.
- capital_required: Set equal to asking_price in USD

CONTENT RULES:
- If content includes a VIDEO TRANSCRIPT, use it as the primary source of truth
- A YouTube video ABOUT a property = type Property (not ContentIdea)
- A YouTube video ABOUT a business = type BusinessIdea
- Only use ContentIdea if the opportunity being evaluated IS the creation of content

GENERAL RULES:
- All monetary values in USD. Conversions: ¥150/USD, CAD 1.38/USD, GBP 0.79/USD, EUR 0.92/USD
- Only include the detail block that matches the type (property_details for Property, etc.)
- Fields marked ALWAYS estimate must never be null — use your market knowledge
- All other fields: only what is stated or logically inferred. Null if truly unknown.
- Return ONLY the JSON object. No preamble, no explanation, no markdown fences.
